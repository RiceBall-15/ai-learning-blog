## 核心概念与挑战

### 为什么Agent需要记忆？

与AI助手对话时，你可能遇到过这些问题：

- **聊了几轮后，它忘记了开头你说的需求**
- **每次重启都要重新解释一遍你的偏好**
- **无法记住上周的决策，导致重复提问**

这不是AI模型本身的问题，而是**上下文管理**的挑战。语言模型的工作记忆（context window）是有限的，通常在32k-200k tokens之间。一旦对话变长，早期信息就会被挤出窗口。

### 记忆系统的核心挑战

| 挑战 | 描述 | 影响 |
|------|------|------|
| **上下文窗口限制** | LLM有固定的token限制 | 早期对话被遗忘 |
| **多轮对话** | 需要记住历史对话 | 无法保持连贯性 |
| **长期知识** | 需要从大量文档中检索相关信息 | 无法访问外部知识 |
| **成本优化** | 只发送最相关的信息给LLM | API调用成本高 |
| **记忆压缩** | 对话历史需要智能压缩 | 丢失重要信息 |

### 记忆系统的本质

根据OpenClaw的实战经验，**文件就是记忆**。所有需要持久化的信息都写入Markdown文件，模型只"记住"写在磁盘上的内容。这套机制看似简单，实际上蕴含了深思熟虑的设计。

## 三层记忆模型

### 记忆架构概览

Agent的记忆系统可以类比人类的记忆结构：

```
Context（工作记忆）
    ↓
Compaction（短期记忆）
    ↓
Memory Files（长期记忆）
```

### 1. 工作记忆（Context）

**定义**：当前会话中模型能"看到"的所有内容，包括系统提示词、对话历史、工具调用结果。

**特点**：
- 受context window限制
- Token用完就需要压缩
- 实时性最强
